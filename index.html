FAQ

How many devices can I create without a subscription?

Users are permitted to create one BleDDevice controller. Each device can contain up to one free control. All control types are available; the limitation applies solely to the quantity. To add additional controls or devices, users may subscribe to the auto-renewable plan, which grants unlimited access to devices, controls, and runs.

Are the configured devices stored remotely?

No. All devices are stored locally on the device. Reinstalling the application will result in the removal of all device configurations. To preserve your devices, it is recommended to share and securely save the configuration data externally.

Is there a limit to the number of controls I can create?

For users with an active subscription, the limit corresponds to the available grid space. The control grid measures 6 x 12 units; the number of controls depends on the selected control type. Users without an active subscription are limited to creating only one BleDControl.

How can I run a configured BleDDevice?

A configured device can be run by simply tapping on its representation within the app.

How do I configure a BleDDevice?

Begin by editing the existing device to add discovery filtering options such as AdvertisementData / ManufacturerData, AdvertisementData / ServiceUUIDs, or Name. Filtering supports regular expression patterns, with a maximum length of 512 characters. After configuring the filters, you can manage its associated BleDControl.

How do I verify if my BleDControls are configured correctly?

A dedicated icon/badge appears in the top-right corner of each control. Controls that are improperly configured display this icon. Additionally, when running the device, only properly configured controls will be active.

Why are some controls absent when I run BleDDevice?

Only controls that are correctly configured will be present and operational during device execution.

I am unsure if I configured a BleDControl properly because I do not observe the expected behavior. What should I do?

There are two additional debugging methods aside from the icon badge:

  • Use the Console app to capture logs related to connectivity and data exchange. Filter logs with bled::cm::. Detailed instructions are provided within the BleDControl configuration view.
  • Simulate BleDControl behavior by sending custom HexStrings via the input field above the control preview. You can also test data transmitted from BleDControl to the connected peripheral.

How can I share a BleDDevice configuration?

Long-press the device to trigger the context menu, then select Distribute / Share.

How can I import a BleDDevice configuration?

Tap on the .beldconfig file and open it within the BleDConfig application.

Is there a possibility to publish devices live so that anyone can run them?

Not yet. I am working on the ability to publish BleDDevice configurations so anyone can trigger them using either QR codes or AppClips. Currently, the only way to share a BleDDevice configuration is with users who have the BleDConfig app installed.

Are there plans to extend the set of available controls?

Yes. In the next app version, new control types are expected to be introduced.
